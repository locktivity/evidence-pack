# Security Policy

## Supported Versions

Evidence Pack Spec v1 (1.x) is the current supported major version line. Security fixes and errata will be published as patch releases (1.x.y).

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅        |

## Reporting a Vulnerability

If you discover a security vulnerability in the Evidence Pack specification, test vectors, samples, or reference implementations, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, email: security@locktivity.com

Include:
1. **Description**
2. **Impact**
3. **Reproduction**
4. **Affected components** (spec section(s) and/or repo paths)
5. **Suggested fix** (optional)

## What to expect

1. **Acknowledgment**: within 48 hours
2. **Assessment**: severity/impact within 7 days
3. **Next steps**: an initial remediation plan or mitigation guidance
4. **Coordination**: coordinated disclosure when appropriate
5. **Credit**: credit in the advisory (unless you prefer anonymity)

### Severity levels (targets)

| Level | Examples | Target time to initial remediation plan |
|-------|----------|------------------------------------------|
| Critical | RCE, key compromise | 24 hours |
| High | Signature bypass, path traversal | 7 days |
| Medium | Info disclosure, DoS | 30 days |
| Low | Minor issues | Next release |
